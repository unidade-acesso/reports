Passo 1: Aceder ao Relatório

A equipa de auditores irá enviar-lhe, por email, o link de acesso ao repositório da auditoria.

No topo da página encontrará:

  • Nome do site ou aplicação móvel
  • URL
  • Nome do proprietário
  • Tipo de selo a que se está a candidatar
  • Link de acesso direto ao relatório
  • Recursos de apoio ao relatório (ex: guias explicativos ou documentação complementar, em caso de dúvidas)

Passo 2: Utilizar o Índice de Conteúdos

Ao clicar no link de acesso, será direcionado para o relatório online.

Já dentro do relatório, encontrará um índice de conteúdos no início da página que lhe permite navegar rapidamente entre as secções através de atalhos diretos.

Passo 3: Analisar a Visão Geral

Consulte a visão geral do relatório, que inclui:

  • Estado por tipo de avaliação
  • Níveis de conformidade por checklist

Passo 4: Explorar a Análise Detalhada

A análise do relatório está organizada em diferentes tipos de avaliação para facilitar a compreensão:

  • Avaliação Automática – realizada com ferramentas automáticas de verificação
  • Avaliação Manual – conduzida por especialistas, dividida em 3 checklists:
    • 10 Aspetos
    • Conteúdo
    • Transação
  • Outras Violações – casos adicionais que não se enquadram nos critérios anteriores, mas que devem ser considerados

Passo 5: Interpretar os Estados de Conformidade

Cada requisito possui etiquetas que indicam o seu estado:

  • OK – Conforme
  • NOK – Não conforme
  • N/A – Não aplicável

Passo 6: Consultar as Evidências no GitHub

Use os botões "Ver detalhes no GitHub" para aceder às evidências da auditoria.

No GitHub encontrará:

  • Título do erro
  • Descrição e provas da análise
  • Sugestões de correção

Passo 7: Criar Conta no GitHub

Para poder comentar e interagir nas evidências da auditoria, é necessário ter uma conta no GitHub.

Siga estes passos para criar a sua conta:

  • Aceda a github.com
  • Clique em "Sign Up" no canto superior direito
  • Preencha o formulário com email, nome de utilizador e palavra‑passe
  • Siga os passos de verificação (pode incluir validação por email)
  • Após criar a conta, inicie sessão e participe nas discussões

Passo 8: Interagir com os Requisitos

Com a sua conta GitHub ativa, pode:

  • Colocar dúvidas
  • Dar feedback sobre as correções
  • Carregar ficheiros como suporte das alterações

Passo 9: Acompanhamento pela Equipa de Auditoria

Um auditor acompanhará cada requisito, colaborando consigo para validar e esclarecer dúvidas.

Passo 10: Iniciar as Correções

Com base nas informações apresentadas no relatório e nas evidências disponíveis no GitHub, inicie o processo de correção dos requisitos não conformes.

À medida que for implementando as correções:

  • Adicione comentários na secção correspondente do GitHub
  • Descreva as alterações feitas e, sempre que possível, anexe evidências (ex: capturas de ecrã, links atualizados, ficheiros modificados)

Os auditores irão acompanhar o progresso, validar as correções efetuadas e reavaliar os requisitos com base na nova informação fornecida.

Final

Agora que conhece todas as funcionalidades do relatório e da plataforma GitHub, está pronto para avançar.

Mãos à obra! Qualquer dúvida, interaja via GitHub.

Bom trabalho!
